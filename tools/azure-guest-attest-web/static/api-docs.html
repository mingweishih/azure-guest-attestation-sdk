🔐 Azure Guest Attestation — REST API Reference

Programmatic access to all attestation features. All endpoints return JSON.

← Web UI API Index (JSON)

Quick Start

The server exposes a REST API at /api/*. All responses share a common JSON envelope: 

{
  "success": true,          // or false on error
  "data": { ... },          // present on success
  "error": "message"        // present on failure
}

Base URL: http://<host>:<port> (or https:// with TLS). For curl with self-signed certs, add -k (or --insecure).

System
GET /api/diagnose System diagnostics & TPM availability ▶

Check OS platform, TPM availability, and CVM type detection. No parameters required.

Example

# curl
curl -s http://localhost:8080/api/diagnose | jq .

# PowerShell
Invoke-RestMethod http://localhost:8080/api/diagnose | ConvertTo-Json

Response

{
  "success": true,
  "data": {
    "os": "linux",
    "arch": "x86_64",
    "tpm_available": true,
    "tpm_status": "TPM opened successfully",
    "cvm_type": "SnpVmReport"
  }
}
Evidence Collection
GET /api/cvm-report Full CVM attestation report ▶

Fetch the full CVM attestation report from the TPM NV index: report header, TEE report (SNP/TDX/VBS), and runtime claims.

Query Parameters

NameTypeDescription
user_data string Optional. Formats: hex:deadbeef, utf8:hello, or raw hex string. ≤64 bytes.

Example

# Without user data
curl -s http://localhost:8080/api/cvm-report | jq .

# With hex user data
curl -s "http://localhost:8080/api/cvm-report?user_data=hex:deadbeef" | jq .

# With UTF-8 user data
curl -s "http://localhost:8080/api/cvm-report?user_data=utf8:hello" | jq .

Response Fields

FieldTypeDescription
report_headerobjectSignature, version, size, request type, status
runtime_claims_headerobjectData size, version, report type, hash type
tee_report_prettystringHuman-readable TEE report fields
tee_report_hexstringRaw TEE report as hex
runtime_claimsobjectParsed runtime claims JSON (if present)
GET /api/tee-report Raw TEE report (SNP/TDX/VBS) ▶

Fetch and parse the raw TEE-specific report. Returns parsed fields and hex encoding.

Query Parameters

NameTypeDescription
user_data string Optional. Same format as /api/cvm-report.

Example

curl -s http://localhost:8080/api/tee-report | jq .data.pretty

Response Fields

FieldTypeDescription
report_typestring"SnpVmReport", "TdxVmReport", or "VbsVmReport"
raw_hexstringRaw report bytes as hex
size_bytesnumberReport size in bytes
prettystringHuman-readable parsed fields

TPM
GET /api/ak-cert Attestation Key certificate (PEM) ▶

Fetch the Attestation Key certificate from the TPM NV index. Returns PEM and base64 encodings.

Example

# Get full JSON
curl -s http://localhost:8080/api/ak-cert | jq .

# Save PEM to file
curl -s http://localhost:8080/api/ak-cert | jq -r .data.pem > ak-cert.pem

# Inspect with openssl
curl -s http://localhost:8080/api/ak-cert | jq -r .data.pem | openssl x509 -text -noout

Response Fields

FieldTypeDescription
pemstringCertificate in PEM format
base64stringDER bytes as base64
size_bytesnumberDER certificate size

GET /api/pcrs PCR values (SHA-1 / SHA-256 / SHA-384) ▶

Read Platform Configuration Register values from the TPM across all available hash banks.

Query Parameters

NameTypeDescription
indices string Optional. Comma-separated PCR indices. Default: 0-23.

Example

# All PCRs
curl -s http://localhost:8080/api/pcrs | jq .

# Specific PCRs
curl -s "http://localhost:8080/api/pcrs?indices=0,1,4,7" | jq .

# Just SHA-256 bank
curl -s http://localhost:8080/api/pcrs | jq '.data.banks["SHA-256"]'

Response Fields

FieldTypeDescription
banksobjectMap of hash algorithm → { "PCR[i]": "hex_digest" }
notesstring[]Errors for unsupported banks (if any)
GET /api/event-log TPM event log with PCR replay ▶

Parse the platform TPM event log and replay PCR values. Reads from standard OS locations (/sys/kernel/security/tpm0/binary_bios_measurements on Linux, C:\Windows\Logs\MeasuredBoot\*.log on Windows).

Example

# Full log
curl -s http://localhost:8080/api/event-log | jq .

# Event count only
curl -s http://localhost:8080/api/event-log | jq .data.event_count

# Replayed PCR values
curl -s http://localhost:8080/api/event-log | jq .data.replayed_pcrs_sha256

Response Fields

FieldTypeDescription
event_countnumberTotal events parsed
eventsarrayArray of { pcr_index, event_type, event_type_name, digest_hex, event_data_size }
replayed_pcrs_sha256objectPCR values replayed from events using SHA-256
Attestation
POST /api/guest-attest Full guest attestation (TPM + TEE) ▶

Perform full guest attestation: collect CVM + device evidence, build an attestation report, and submit to a provider. Returns a JWT token (MAA) or echo (loopback).

Request Body (JSON)

FieldTypeDefaultDescription
provider string "loopback" "loopback" (echo) or "maa" (Azure Attestation)
endpoint string — MAA endpoint URL (required when provider is "maa")
client_payload string — Optional JSON string of key-value pairs
pcr_indices number[] OS default PCR indices to include in attestation
decode_token boolean false Decode the returned JWT and include header/payload

Example

# Loopback (echo) — for testing
curl -s -X POST http://localhost:8080/api/guest-attest \
  -H "Content-Type: application/json" \
  -d '{"provider":"loopback","decode_token":true}' | jq .

# MAA attestation
curl -s -X POST http://localhost:8080/api/guest-attest \
  -H "Content-Type: application/json" \
  -d '{
    "provider": "maa",
    "endpoint": "https://sharedweu.weu.attest.azure.net",
    "decode_token": true
  }' | jq .

# PowerShell
$body = @{
    provider = "maa"
    endpoint = "https://sharedweu.weu.attest.azure.net"
    decode_token = $true
} | ConvertTo-Json
Invoke-RestMethod -Method Post -Uri http://localhost:8080/api/guest-attest `
  -ContentType "application/json" -Body $body | ConvertTo-Json -Depth 10

Response Fields

FieldTypeDescription
pcrsobjectPCR values included in the attestation
request_jsonstringRaw JSON request sent to the provider
encoded_requeststringBase64-encoded request
tokenstringJWT token returned by the provider
decryptedbooleanWhether the token was decrypted from JWE envelope
token_headerobjectDecoded JWT header (when decode_token=true)
token_payloadobjectDecoded JWT payload / claims (when decode_token=true)
POST /api/tee-attest TEE-only attestation (no TPM/PCR) ▶

Perform TEE-only platform attestation without TPM/PCR evidence. Sends SNP report + VCEK chain or TDX quote directly to an MAA platform endpoint.

Request Body (JSON)

FieldTypeDefaultDescription
endpoint string "https://sharedweu..." MAA platform endpoint URL
decode_token boolean false Decode the returned JWT

Example

curl -s -X POST http://localhost:8080/api/tee-attest \
  -H "Content-Type: application/json" \
  -d '{
    "endpoint": "https://sharedweu.weu.attest.azure.net",
    "decode_token": true
  }' | jq .

Response Fields

FieldTypeDescription
request_jsonstringRaw JSON request sent to MAA
tokenstringJWT token returned by MAA
token_headerobjectDecoded JWT header
token_payloadobjectDecoded JWT payload / claims
Tools
POST /api/parse-token Decode a JWT attestation token ▶

Decode a JWT attestation token into its header and payload. No TPM required — pure parsing. Useful for inspecting tokens obtained from other sources.

Request Body (JSON)

FieldTypeDescription
token string The JWT string to decode (required)

Example

curl -s -X POST http://localhost:8080/api/parse-token \
  -H "Content-Type: application/json" \
  -d '{"token":"eyJhbGciOi..."}' | jq .

Response Fields

FieldTypeDescription
headerobjectJWT header (alg, kid, typ, …)
payloadobjectJWT payload / claims
TDX Endorsement (THIM)
GET /api/endorsement/mrtds List known TDX MRTDs from Azure THIM ▶

Queries the Azure Trusted Hardware Identity Management (THIM) service for all known endorsed TDX MRTD values (SHA-384 measurement hashes).

Query Parameters

ParamTypeDescription
regionstringAzure region (default: westus)
curl -s http://localhost:8080/api/endorsement/mrtds | jq .
curl -s http://localhost:8080/api/endorsement/mrtds?region=eastus | jq .

Response Fields

FieldTypeDescription
countnumberNumber of MRTDs returned
mrtdsstring[]Array of uppercase hex MRTD values (96 chars each)
GET /api/endorsement/tdx/:mrtd Fetch TDX endorsement (CoRIM) for a specific MRTD ▶

Fetches the COSE/CoRIM endorsement binary for a specific MRTD value from THIM. The response includes the base64-encoded endorsement payload.

Path Parameters

ParamTypeDescription
mrtdstring96-character uppercase hex MRTD value

Query Parameters

ParamTypeDescription
regionstringAzure region (default: westus)
curl -s http://localhost:8080/api/endorsement/tdx/6954FB6A...E37E8 | jq .

Response Fields

FieldTypeDescription
mrtdstringThe MRTD value (uppercase hex)
content_typestringContent type from THIM (e.g. application/cose)
size_bytesnumberSize of the endorsement payload
base64stringBase64-encoded endorsement binary
GET /api/endorsement/from-platform Auto-detect MRTD from platform TDX report ▶

Reads the TDX report from the platform TPM, extracts the mr_td field, and fetches the corresponding endorsement from THIM. Only works on TDX CVMs.

Query Parameters

ParamTypeDescription
regionstringAzure region (default: westus)
curl -s http://localhost:8080/api/endorsement/from-platform | jq .

Response Fields

FieldTypeDescription
mrtdstringExtracted MRTD from the platform report
content_typestringContent type from THIM
size_bytesnumberSize of the endorsement payload
base64stringBase64-encoded endorsement binary
